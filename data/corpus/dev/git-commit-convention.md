# Git 提交信息规范

## 一、目的

统一的提交信息便于自动生成 changelog、快速定位变更、支持语义化版本。
本规范基于 Conventional Commits 约定。

## 二、提交信息格式

```
<type>(<scope>): <subject>

<body>

<footer>
```

- **type**：变更类型（见下）。
- **scope**：影响范围，可选，如模块名。
- **subject**：简短描述，不超过 50 字，结尾不加句号。
- **body**：详细说明变更动机与影响，可选。
- **footer**：关联 issue（如 `Closes #123`）或破坏性变更说明。

## 三、type 取值

| type | 含义 |
| ---- | ---- |
| feat | 新功能 |
| fix | 缺陷修复 |
| docs | 文档变更 |
| style | 格式调整（不影响逻辑） |
| refactor | 重构（非新功能非修复） |
| test | 测试相关 |
| chore | 构建/工具链/依赖等杂项 |
| perf | 性能优化 |

## 四、强制要求

- 一次提交只做一件事，避免混杂多个无关变更。
- 破坏性变更必须在 footer 用 `BREAKING CHANGE:` 显式标注。
- 禁止提交 `update`、`fix bug`、`修改` 等无意义信息。
- 禁止在提交信息中包含敏感信息（密码、密钥、内网地址）。
