# 日志规范

## 一、日志级别

| 级别 | 用途 |
| ---- | ---- |
| ERROR | 影响功能的错误，需要告警与人工介入 |
| WARN | 潜在问题，暂不影响主流程 |
| INFO | 关键业务节点、状态变更 |
| DEBUG | 调试细节，生产环境默认关闭 |

禁止用 print 输出日志，统一使用日志框架。

## 二、日志内容

- 每条日志须包含时间戳、级别、模块、traceId（链路追踪 ID）。
- 错误日志须包含完整堆栈与上下文参数。
- 禁止记录密码、密钥、完整身份证号等敏感信息，必要时脱敏。

## 三、结构化日志

- 生产环境采用 JSON 结构化日志，便于采集与检索。
- 关键字段标准化：`level`、`ts`、`service`、`trace_id`、`msg`。

## 四、链路追踪

- 一次请求全链路使用同一 traceId，跨服务透传。
- traceId 在入口（网关/控制器）生成，写入日志与响应头。

## 五、留存与告警

- 生产日志至少留存 30 天，审计日志留存 180 天。
- ERROR 级别日志接入告警，5 分钟内通知值班人员。
