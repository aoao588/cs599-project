# 安全开发规范

## 一、密钥与凭据管理

- 严禁在代码、配置文件、日志中硬编码密码、API Key、Token。
- 所有密钥通过环境变量或密钥管理服务（KMS / Vault）注入。
- 密钥须定期轮换，泄露后立即吊销并排查影响范围。

## 二、输入校验

- 所有外部输入默认不可信，须做白名单校验。
- 防 SQL 注入：使用参数化查询，禁止字符串拼接 SQL。
- 防 XSS：输出到页面的用户内容须转义。
- 文件上传须校验类型、大小，存储路径与执行目录隔离。

## 三、认证与授权

- 密码须加盐哈希存储（bcrypt / argon2），禁止明文或可逆加密。
- 会话 Token 须设置过期时间，支持主动吊销。
- 遵循最小权限原则，按角色授予最小必要权限。

## 四、数据保护

- 敏感数据（身份证、手机号、银行卡）传输与存储须加密。
- 日志中的敏感字段须脱敏，如手机号显示为 `138****1234`。
- 数据导出须审批并留痕。

## 五、依赖与漏洞

- 定期扫描第三方依赖漏洞（SCA），高危漏洞须在 48 小时内修复。
- 禁止引入未经评估的来源不明的依赖。
