# 测试规范

## 一、测试分层

| 层级 | 范围 | 占比目标 |
| ---- | ---- | -------- |
| 单元测试 | 单个函数/类，隔离外部依赖 | 70% |
| 集成测试 | 模块间协作、数据库、缓存 | 20% |
| 端到端测试 | 完整业务流程 | 10% |

遵循测试金字塔，单元测试为主，端到端测试为辅。

## 二、覆盖率要求

- 整体行覆盖率不得低于 70%，新增代码覆盖率不得低于 80%。
- 核心模块（支付、鉴权、计费）覆盖率不得低于 90%。
- 覆盖率由 CI 强制门禁，不达标无法合并。

## 三、单元测试约定

- 测试命名遵循 `test_<被测方法>_<场景>_<预期>`。
- 每个测试只验证一个行为，遵循 AAA 模式（Arrange-Act-Assert）。
- 外部依赖（网络、数据库、时间）必须 mock，保证测试可重复、快速。
- 禁止测试之间相互依赖执行顺序。

## 四、测试数据

- 使用工厂方法或 fixture 构造测试数据，禁止依赖生产数据。
- 测试结束须清理产生的数据，保证幂等。

## 五、回归测试

- 每次修复 bug 必须补充能复现该 bug 的测试用例。
- 发布前须执行完整回归测试套件。

## 六、解释权
本规范由质量保障组维护，最近更新：2025-07-15。
