# 值班与故障响应规范

## 一、值班安排

- 核心服务实行 7×24 小时值班，按周轮换，每周一上午交接。
- 值班人员须保证手机畅通，30 分钟内可远程接入。
- 值班期间不得单独执行高风险变更。

## 二、故障分级

| 等级 | 定义 | 响应时限 |
| ---- | ---- | -------- |
| P0 | 核心服务不可用、资金损失、数据泄露 | 立即响应，15 分钟内拉起应急 |
| P1 | 重要功能受损，影响大量用户 | 30 分钟内响应 |
| P2 | 局部功能异常，有规避方案 | 2 小时内响应 |
| P3 | 轻微问题，不影响主流程 | 下个工作日处理 |

## 三、故障处理流程

1. 发现/收到告警，确认故障等级。
2. P0/P1 立即在应急群同步，拉相关方。
3. 优先止血（回滚、降级、限流），而非定位根因。
4. 恢复后保留现场，便于复盘。

## 四、事后复盘

- P0/P1 故障须在 48 小时内提交复盘报告（Postmortem）。
- 复盘对事不对人，聚焦改进措施与防止复发。
- 复盘产出的改进项须有负责人与截止时间，纳入跟踪。

## 五、解释权
本规范由 SRE 团队维护，最近更新：2025-10-18。
