# Product Spec — EnterpriseDocAgent

> **版本**：v0.1 · **日期**：2026-06-02 · **状态**：Draft

---

## 1. 背景与问题

中型软件公司内部知识高度碎片化：研发规范散落在 Confluence/Wiki/Markdown，HR 制度在 PDF，财务/项目数据在数据库。员工日常会问：

- "公司年假怎么算？我刚入职 8 个月能休几天？"
- "我们的代码评审有哪几条强制规则？"
- "最近三个月研发部出差总金额是多少？"

现状的两类方案都不够好：

| 方案 | 痛点 |
|---|---|
| 传统全文搜索（ES / 站内搜索） | 关键词匹配不懂语义；结果是文档列表，不是答案 |
| 朴素 RAG（单轮检索 + 生成） | 一次检索定胜负；遇到口语化提问、跨库提问、需要计算的提问就答非所问；无引用，无法核对 |

---

## 2. 产品定位

**EnterpriseDocAgent** = 一个**会规划、会反思、带引用**的企业知识问答 Agent。

它和「朴素 RAG」的本质区别：**把"检索"从一次性动作变成一个由 LLM 驱动的多步决策过程**。

```
朴素 RAG：  Query ──→ Retrieve ──→ Generate
Agentic： Query ─→ Plan ─→ Retrieve ─→ Grade ─→（不够好就回到 Plan）─→ Generate ─→ Reflect
```

---

## 3. 目标用户与场景

| 角色 | 典型场景 | 价值 |
|---|---|---|
| 普通员工 | "请假/报销/规范"类高频咨询 | 30 秒拿到带引用的答案，不再骚扰 HR |
| 新员工 | 入职第一周的"什么都不懂" | 自助答疑，缩短上手时间 |
| 技术负责人 | "我们对 XX 的规范是什么" | 跨多个 Wiki 一次性聚合 |
| 业务主管 | "最近 N 个月 XX 数据" | 自然语言查表，不必学 SQL |

## 4. 用户故事（MVP 范围）

### US-01 文档问答（必做）
> 作为员工，我向 Agent 提问 HR 或研发规范类问题，期望得到答案和**可追溯的来源**（文档名 + 段落）。

**验收**：
- 提问"年假怎么算" → 返回正确政策文本 + 引用 `hr/leave-policy.md`
- 答案中每个事实点都有 `[1]` `[2]` 角标，文末列出对应源

### US-02 跨库提问（必做）
> 作为员工，当我的问题涉及多个知识库时，Agent 能自动判断需要查哪些，并合并结果。

**验收**：
- "新入职员工的报销流程和额度是多少" → 同时检索 HR + 财务规范

### US-03 检索不足时自动重检（必做）
> 作为员工，当首轮检索结果不充分时，Agent 应改写 Query 重试，而不是凑答案。

**验收**：
- 对模糊提问"这事儿合规吗"先返回澄清问题，或自动改写为更具体的 Query 再检索
- 反思循环最多 `MAX_REFLECT_ROUNDS=2` 轮，避免死循环

### US-04 结构化数据查询（增强）
> 作为主管，我用自然语言提数据问题，Agent 调 `sql_query` 工具查 DuckDB 并给出结果。

**验收**：
- "Q1 研发部出差总额" → Agent 生成 SQL → 查 DuckDB → 返回数值

### US-05 拒绝幻觉（必做）
> 作为员工，当知识库没有相关信息时，Agent 必须明确说"未找到"，**不许编造**。

**验收**：
- 提问知识库外的内容 → 回答包含"未在知识库中找到相关依据"

## 5. 非功能性需求

| 维度 | 目标 |
|---|---|
| 响应时延 | P50 < 6s，P95 < 12s（含两轮反思） |
| 引用准确率 | ≥ 90%（Ragas `context_precision`） |
| 答案忠实度 | ≥ 85%（Ragas `faithfulness`，无幻觉） |
| 答案相关性 | ≥ 85%（Ragas `answer_relevancy`） |
| 安全 | API Key 全部走环境变量；输入做基础注入过滤 |
| 可观测 | 每次请求生成一条 LangSmith trace，可逐节点回放 |

---

## 6. 范围边界（MVP 不做）

- 多租户与权限隔离
- 文档实时增量同步（先做手动 `ingest`）
- 复杂 SQL（聚合可以，多表 JOIN 暂不保证）
- 多智能体编排（架构预留，Final 视情况升级）

---

## 7. 成功指标

| 指标 | MVP（06.08） | Final（06.22） |
|---|---|---|
| Golden QA 集大小 | 10 条 | 30 条 |
| 答案正确率（人工评） | ≥ 70% | ≥ 85% |
| Ragas faithfulness | 跑通即可 | ≥ 0.85 |
| 端到端可用性 | 本地 CLI 可跑 | 云端 URL 可访问 |

## 8. 演进路线

- **v0.1（MVP）**：单 Agent + Chroma + DeepSeek + 通义 Embedding，CLI 可用
- **v0.2**：接入 MCP Server（vector_search / sql_query 标准化）
- **v0.3**：Streamlit UI + Docker Compose + 阿里云 ECS 部署
- **v0.4（可选）**：拆分为 Router / Retriever / Critic 多 Agent
